# Save Lord — Roguelite CLI Tactical RPG

> **游戏名称**：Save Lord（谐音 SSL — Save & Load）
> **目标**：AI 智能体在阅读本提示词后，**一次性完整构建**该游戏，无需后期调整。
> **目标用户**：资深玩家、程序员、人工智能（AI 本身）
> **运行环境**：命令行界面（CMD / 任何 CLI），使用自定义等宽终端（字符为正方形，大小为正常字符的 1/4）
> **语言**：仅英语，所有物品/技能/角色名称为英文全拼
> **编程语言**：Rust

---

## 一、游戏核心概念

### 1.1 游戏名称与核心玩法
- 游戏名：**Save Lord**

  

- **核心玩法**：游戏极其困难，鼓励玩家通过**不停保存和加载（Save & Load）** 来推进

- 玩家可以回溯**任何一个操作**，回溯多久都行

### 1.2 种子系统（Seed System）
- 每局游戏开始前生成一个**固定的随机数种子（Seed）**
- 该种子决定本局游戏中**所有随机事件**的结果
- 玩家**不知道**种子是什么
- 这意味着所有随机事件从一开始就已经被确定，只是玩家不知道而已

### 1.3 存档系统
- **完善的自动存档机制**
- 可以回溯**任何一个操作**，时间不限
- **极其良好的读存档机制**：存档内容对玩家完全透明，玩家可以看懂所有存档/读档的内容
- 存档以可读格式存储（如 JSON），方便玩家理解

### 1.4 输入方式
- 仅支持**指令输入**（类似正常 CMD），不读取单个按键
- 用户输入指令后回车执行

### 1.5 显示与 API
- 终端显示纯文本 UI，字符为**等宽正方形**（1/4 正常字符大小）
- 提供 **API 接口**，允许 AI 接入并读取当前纯文本 UI 状态
- AI 可以通过 API 看到游戏当前的界面并参与交互

### 1.6 局外养成系统
- 每局游戏结束后根据**分数**获得经验值（XP）
- XP 可用于**解锁更多物品/角色/技能**
- 形成"多周目"成长循环

## 三、领主系统（Lord System）

### 3.1 领主（Lord）
- 领主是玩家**开局拥有**的单位
- 领主**不能下阵**（永远在队伍中）
- **领主死亡 = 全队死亡 = 游戏失败**
- 领主是队伍的核心，保护领主是最高优先级

### 3.2 胜利条件
- **唯一的胜利方法**：击杀敌方的领主
- 无论消灭多少小兵，只要敌方领主存活就不算赢

### 3.3 领主升级
- 领主有等级，最高 **5 级**
- 升级方式（**不能通过抽卡升星升级**）：
  - **事件**（Event）
  - **特殊遗物**（Special Relic）
  - **通过的关卡数量**（自动升级）
- 领主升级效果：提升领主自身属性 + 增加可用的队伍数量

### 3.4 队伍数量
- 开局拥有 **4 个队伍**
- 最多拥有 **10 个队伍**
- 增加队伍数量的唯一途径：**升级领主**
- 队伍数量 = 领主等级决定的可用队伍上限

## 四、地图系统（类《杀戮尖塔》）

### 4.1 结构
- 共 **4 层（Floor）**
- 每层包含 **1 个 Boss + 10 个节点**
- 玩家必须经过 **10 个节点** 后才能挑战 Boss

### 4.2 节点类型
- **火堆（Campfire）** — 恢复/强化/升级队伍
- **商店（Shop）** — 购买物品
- **怪物（Monster）** — 战斗
- **事件（Event）** — 随机事件

### 4.3 路径选择
- 树状图结构，类似杀戮尖塔
- 玩家通过选择**就近节点**进行分叉
- 每个节点连接后续 1~3 个可选节点

---

## 五、火堆系统（Campfire）

火堆节点提供以下功能：

### 5.1 复活（Revive）
- **复活所有队伍中在子战斗中死亡的角色**
- 子战斗中死亡的角色在战斗结束后保持死亡状态，只有通过火堆才能复活

### 5.2 升级队伍（Upgrade Team）
- 增加队伍的人口上限（每个队伍最多 6 人）
- 提升队伍整体实力

### 5.3 其他升级途径
除了火堆，还可以通过以下方式升级队伍：
- **事件**（Event 节点）
- **特殊遗物**（Relic）

## 六、战斗系统

### 6.1 速度条（Speed Bar）
- 类似《崩坏：星穹铁道》/《阴阳师》的速度条机制
- 所有单位在速度条上排列
- 速度快的单位行动频率更高
- 速度条显示在界面**左侧**

### 6.2 战棋地图（Tactical Grid）
- 类似《梦幻模拟战》/《火焰之纹章》的格子战棋
- 角色可以在网格上移动
- 地形影响移动和战斗

### 6.3 单位系统
- 可操纵单位称为**队伍（Team）**
- 每个队伍包含多个**角色（Character）**
- 角色有**排位（Position/Rank）**，排位决定技能发挥方式
- 排位可以移动

### 6.4 队伍人数
- 初始：**2 人**（最少）
- 可通过火堆/事件/遗物增加
- 最多 **6 人**

### 6.5 角色唯一性约束
- **角色不能重复上阵**：同一个角色不能出现在多个队伍中，即使是不同队伍也不行
- 整个游戏中每个角色只能有一个实例被部署

### 6.6 敌人 AI 行为
- 敌人队伍会**主动追击**玩家队伍
- 敌人**优先攻击领主**所在的队伍
- 多个敌人队伍之间会协作包围玩家

## 七、技能系统

技能按**顺位（Priority Order）** 解锁和使用，顺序如下：

### 7.1 普通攻击（Normal Attack）— 顺位 1
- 速度条轮到该单位时可用
- **不消耗任何资源**
- 每次普攻获得 **1 个技能点（Skill Point）**

### 7.2 移动（Move）— 顺位 2（被动技能）
- 速度条轮到该单位时可用
- 消耗**移动力（Movement Points）**
- 地形系统（类似《火焰之纹章》）：
  - 不同单位经过特定格子消耗不同移动力
  - **障碍（Obstacle）**：仅飞行单位可穿过
  - **水域（Water）**：仅游泳/飞行单位可穿过
  - **树林（Forest）**：特定单位速度降低
  - 移动力耗尽后无法继续移动

### 7.3 技能点技能（Skill Point Skill）— 顺位 3
- 消耗 **1 个技能点**
- 技能点由**普通攻击获得**，所有队伍共享
- 与普通攻击**二选一**：速度条轮到该单位时，只能选择普攻或技能点技能之一
- 大部分情况下每回合只能使用一次

### 7.4 CD 技能（Cooldown Skill）— 顺位 4
- 有冷却回合数（CD），通过**经过的回合数**减少
- **开局默认可用**
- **可以插队**：无论速度条是否轮到该单位，都可以使用

### 7.5 能量技能（Energy Skill）— 顺位 5
- 释放任何技能（普攻/技能点技能/CD技能/终结技）都会获得**能量**
- 能量条满后可释放
- **可以插队**：无论速度条是否轮到该单位，都可以使用

### 7.6 终结技能（Ultimate Skill）— 顺位 6
- **每场战斗仅可使用一次**
- **可以插队**：不论是否轮到该单位，都可以立即使用

## 八、战斗判定系统（子战斗系统）

当两个队伍发生战斗时，进入**子战斗（Sub-Battle）** 模式。这是一个独立于战棋地图的回合制对决。

### 8.1 触发条件
- 当攻击方队伍在战棋地图上**发起攻击**时触发
- 或防守方队伍**受到攻击**时触发

### 8.2 子战斗流程总览

```
[战棋地图] 队伍A攻击队伍B
    ↓
[进入子战斗]
    ├── 阶段一：准备阶段（双方队伍排列）
    ├── 阶段二：主动阶段（Active Phase）
    │   ├── 进攻方先手（先发优势生效）
    │   └── 防守方后手
    ├── 阶段三：被动阶段（Passive Phase）
    │   ├── 进攻方被动触发
    │   └── 防守方被动触发
    └── 阶段四：结算阶段
        ↓
[返回战棋地图] 根据结果更新状态
```

### 8.3 队伍排列（Formation）

每个队伍在子战斗中按**排位（Position）** 排列：

```
[前排 Frontline]     [中排 Midline]     [后排 Backline]
   角色A                 角色C               角色E
   角色B                 角色D               角色F
```

- **前排**：更容易被攻击，但近战伤害加成
- **中排**：均衡位置
- **后排**：更难被攻击，远程伤害加成
- 角色可以在子战斗开始前调整排位（消耗一次行动机会）

### 8.4 每个角色的属性

```
Character {
    hp: i32,                    // 当前生命值
    max_hp: i32,                // 最大生命值
    atk: i32,                   // 攻击力
    def: i32,                   // 防御力
    spd: i32,                   // 速度（决定子战斗内行动顺序）
    star_level: u32,            // 星级（1/2/3... 通过升星提升）
    position: Position,         // 排位 (Front/Mid/Back)
    active_points: u32,         // 技能点（主动发出）
    passive_points: u32,        // 被动点（被动发出）
    active_skills: Vec<Skill>,  // 主动技能列表
    passive_skills: Vec<Skill>, // 被动技能列表
    equipment: Vec<Equipment>,  // 装备列表（最多2个）
    passive_abilities: Vec<PassiveAbility>, // 装备提供的被动能力（非技能点系统）
    buffs: Vec<Buff>,           // 增益/减益状态
}
```

### 8.5 角色基础属性

所有玩家可获得的角色拥有**固定的基础属性**：

| 属性 | 基础值 |
|------|-------|
| ATK（攻击力） | 100 |
| HP（生命值） | 100 |
| SPD（速度） | 100 |

- 这些基础属性可以通过**装备**的被动能力进行改变
- 升星也会提升属性
- **敌方角色的属性不固定**，由 AI 自行设计

### 8.6 主动阶段（Active Phase）详细规则

**主动技能 vs 被动技能的区别**：
- **主动技能**：速度条轮到该单位时触发，由玩家/ AI 选择使用
- **被动技能**：满足编程设定的条件时**自动触发**，不需要速度条轮到

**行动顺序**：
1. 进攻方所有角色按 **spd（速度）从高到低** 依次行动
2. 然后防守方所有角色按 **spd 从高到低** 依次行动
3. 进攻方拥有**先发优势**：进攻方造成的伤害 × 1.2，受到的伤害 × 0.8

**每个角色的行动**：
1. 消耗 **1 个 active_point**
2. 从该角色的 active_skills 中选择一个技能使用
3. 技能效果立即生效（伤害/治疗/增益/减益/位移等）
4. 如果角色 active_points 耗尽，跳过该角色

**技能目标选择规则**（按优先级）：
1. 技能本身指定目标（如"攻击前排"）
2. 默认优先攻击**前排**角色
3. 如果前排角色全部阵亡，攻击**中排**
4. 如果中排全部阵亡，攻击**后排**
5. 某些技能可以"穿透"直接攻击后排

**伤害计算公式**：
```
base_damage = attacker.atk - target.def
if base_damage < 1 { base_damage = 1 }

position_modifier:
  - 前排攻击前排：× 1.0
  - 前排攻击中排：× 0.7
  - 前排攻击后排：× 0.4
  - 中排攻击前排：× 1.1
  - 中排攻击中排：× 1.0
  - 中排攻击后排：× 0.7
  - 后排攻击前排：× 1.2
  - 后排攻击中排：× 1.0
  - 后排攻击后排：× 1.0

first_strike_modifier:
  - 进攻方攻击时：× 1.2
  - 进攻方被攻击时：× 0.8
  - 防守方无修正

star_modifier:
  - 每比对方高 1 星：× 1.1
  - 每比对方低 1 星：× 0.9

critical_hit: 5% 概率，× 2.0 伤害

final_damage = base_damage × position_modifier × first_strike_modifier × star_modifier × (critical ? 2.0 : 1.0)
```

### 8.7 被动阶段（Passive Phase）详细规则

**被动技能不是按速度条触发的，而是满足编程条件时自动触发。**

主动阶段结束后进入被动阶段：

1. 每个角色按 **spd 从高到低** 依次检查被动触发条件
2. 消耗 **1 个 passive_point**
3. 从 passive_skills 中选择满足条件的被动技能触发
4. 被动技能类型及触发条件示例：
   - **反击（Counter）**：触发条件 = 本回合受到过攻击 → 对攻击者造成伤害
   - **减伤（Guard）**：触发条件 = 自身 HP 低于 30% → 减少下一次受到的伤害
   - **治疗（Heal）**：触发条件 = 队友 HP 低于 50% → 治疗该队友
   - **闪避（Dodge）**：触发条件 = 即将受到致命伤害 → 概率闪避
   - **净化（Cleanse）**：触发条件 = 自身有 debuff → 移除所有负面状态
   - **嘲讽（Taunt）**：触发条件 = 队友即将被攻击 → 强制敌人攻击自己
   - **护卫（Protect）**：触发条件 = 相邻队友被选为目标 → 代替承受伤害
5. 如果角色 passive_points 耗尽，跳过
6. 如果没有任何被动满足触发条件，该角色跳过

### 8.8 子战斗中的死亡与复活

- 在子战斗中死亡的角色，**不会在战斗结束后自动复活**
- 子战斗中可以通过**角色技能**进行复活（在判定中复活）
- 但子战斗结束后，死亡的角色保持死亡状态
- **只有通过火堆（Campfire）才能复活子战斗中死亡的角色**

### 8.9 装备系统（子战斗中的被动能力）

装备（Equipment）是战斗中掉落的重要物品，用于增强角色。

**装备规则**：
- 装备可以为角色增加**被动能力（Passive Ability）**，但**不能增加主动技能**
- 这里的"被动能力"是纯粹的属性/效果增强，**不是**使用被动技能点触发的那个系统
- 每个角色最多装备 **2 个装备**
- 装备**没有稀有度**，**没有星级**
- **装备可以重复**：多个角色可以装备同名的装备

**被动能力示例**：
- 攻击力 +10%
- 防御力 +15%
- 速度 +5
- 受到治疗量 +20%
- 暴击率 +3%
- 移动力 +1
- 对特定地形适应性提升
- 每回合恢复 5% HP

### 8.10 子战斗结束条件

满足以下任一条件时子战斗结束：

1. **一方全灭**：某方所有角色 HP ≤ 0 → 另一方获胜
2. **双方点数耗尽**：所有角色的 active_points 和 passive_points 均为 0
   - 比较双方剩余 HP 百分比总和
   - 高者获胜
   - 平局则进攻方判负（进攻失败）
3. **一方撤退**：玩家可以选择撤退（损失部分兵力）

### 8.11 子战斗结果对战棋地图的影响

- 获胜方：保留剩余 HP 回到战棋地图
- 失败方：队伍全员 HP 归零，从战棋地图上移除
- 撤退方：所有角色 HP 减半，退回攻击前的位置

### 8.12 攻击顺序与逻辑的编程要求

攻击顺序和攻击方式由**复杂编程逻辑**决定，需要实现一个**决策引擎（Decision Engine）**，包含最多 100 个判断条件，覆盖以下维度：

```
判断维度示例：
├── 目标选择
│   ├── 优先攻击血量最低的敌人
│   ├── 优先攻击排位最前的敌人
│   ├── 优先攻击特定职业/类型的敌人
│   ├── 优先攻击有 debuff 的敌人
│   └── 优先攻击正在蓄力的敌人
├── 技能选择
│   ├── 对多个敌人时使用 AOE 技能
│   ├── 对单体时使用高伤害技能
│   ├── 队友血量低时使用治疗技能
│   ├── 自身血量低时使用防御/回复技能
│   ├── 敌人有 buff 时使用驱散技能
│   └── 有击杀机会时使用终结技
├── 被动触发
│   ├── 受到攻击时优先反击
│   ├── 队友被攻击时优先护卫
│   ├── 血量低时优先闪避/减伤
│   └── 有 debuff 时优先净化
├── 排位策略
│   ├── 前排血量低时与中排换位
│   ├── 后排被穿透攻击时换位到前排
│   └── 特定技能需要特定排位才能释放
└── 特殊状态判断
    ├── 敌人正在准备大招时优先打断
    ├── 自身有增伤 buff 时优先输出
    ├── 敌人有减伤 buff 时等待或驱散
    └── 即将轮到 Boss 行动时优先防御
```

## 九、遗物系统（Relic System）

### 9.1 遗物概述
- 遗物（Relic）类似于《杀戮尖塔》中的遗物
- 遗物是**稀有**物品，不是每次都能获得
- 遗物提供强大的全局效果

### 9.2 获取途径
遗物可以通过以下方式获得（**可能获得，也可能不获得**）：
- **事件**（Event 节点）
- **商店**（Shop 节点 — 购买）
- **战斗结束后**（Boss 战或特定战斗掉落）

### 9.3 遗物效果示例
- 所有队伍攻击力 +15%
- 所有队伍防御力 +15%
- 每场战斗开始时获得 3 个技能点
- 火堆效果翻倍
- 商店物品打 8 折
- 每层开始时恢复所有角色 20% HP
- 速度条速度 +10%
- 子战斗中额外获得 2 个 active_point
- 领主获得额外护盾

## 十、事件系统（Event System）

### 10.1 事件节点
- 所有事件节点都是**选择事件**（有分支选择）
- 所有事件**全部都是正面事件**
- 但如果选择不好，可能会有**负面效果**
- **正面效果绝对大于负面效果**，但负面效果仍然存在

### 10.2 事件示例

**事件 1：Ancient Treasure（古代宝藏）**
> 你发现了一个古老的宝箱。
> 
> 选择 A：谨慎开启 → 获得 3 件随机装备 + 可能触发陷阱（一个角色损失 10% HP）
> 选择 B：暴力砸开 → 获得 5 件随机装备 + 必定触发陷阱（一个角色损失 20% HP）
> 选择 C：绕道离开 → 无事发生

**事件 2：Wandering Merchant（流浪商人）**
> 一个神秘的商人愿意与你交易。
> 
> 选择 A：用金币购买 → 获得 1 个随机遗物（消耗 50 金币）
> 选择 B：以物易物 → 用 2 件装备交换 1 件更好的装备
> 选择 C：拒绝交易 → 无事发生

**事件 3：Healing Spring（治愈之泉）**
> 你发现了一处神奇的泉水。
> 
> 选择 A：饮用泉水 → 所有角色恢复 50% HP
> 选择 B：浸泡全身 → 所有角色恢复 100% HP，但有一个角色中毒（每回合损失 5% HP，持续 3 回合）
> 选择 C：采集泉水 → 获得 2 瓶药水

**事件 4：Training Ground（训练场）**
> 你发现了一个废弃的训练场。
> 
> 选择 A：让领主训练 → 领主获得经验值（升级进度 +20%）
> 选择 B：让队伍训练 → 所有队伍中的角色获得临时 ATK +20%（持续本层）
> 选择 C：休息 → 无事发生

**事件 5：Mysterious Altar（神秘祭坛）**
> 一座古老的祭坛散发着光芒。
> 
> 选择 A：献祭 1 个角色 → 获得 1 个强力遗物（角色被献祭后在本局中永久消失）
> 选择 B：献祭 10% 最大 HP → 所有角色 ATK +10%（持续本局）
> 选择 C：无视 → 无事发生

**事件 6：Supply Cache（补给仓库）**
> 你找到了一个补给仓库。
> 
> 选择 A：拿取食物 → 获得 3 份食物
> 选择 B：拿取药水 → 获得 3 瓶药水
> 选择 C：全部拿走 → 获得 2 份食物 + 2 瓶药水，但触发警报（随机一个队伍被敌人追击）

## 十一、Boss 系统

### 11.1 Boss 概述
- 每层有一个 Boss 节点
- Boss 队伍人数**不固定**
- Boss 队伍中有一个**很强的单位**被称为 Boss

### 11.2 Boss 特性
- 部分 Boss 拥有**阶段转换（Phase Transition）**，部分没有
- 阶段转换示例：HP 低于 50% 时进入第二阶段，获得新技能/属性提升
- Boss 的属性和技能由 AI 自行设计，应明显强于普通敌人

### 11.3 Boss 行为
- Boss 队伍会**主动追击**玩家
- Boss **优先攻击领主**
- Boss 队伍中的小兵会保护 Boss

## 十二、角色池（Character Pool）

### 12.1 角色数量
- 游戏中共有 **1000 名不同的角色**
- 所有角色通过抽卡获得

### 12.2 角色模板
所有角色的基础属性**固定**：

```
Character Template {
    name: String,       // 英文全拼名称（唯一）
    atk: 100,           // 基础攻击力
    max_hp: 100,        // 基础最大生命值
    spd: 100,           // 基础速度
}
```

- 所有角色基础属性相同（ATK=100, HP=100, SPD=100）
- 角色之间的区别在于**技能不同**
- 属性可以通过**装备**的被动能力改变
- 升星会提升属性

### 12.3 角色技能生成
AI 需要为 1000 个角色生成不同的技能组合。每个角色拥有：
- 1 个普通攻击技能（顺位 1）
- 1 个移动类型（顺位 2）
- 0~1 个技能点技能（顺位 3）
- 0~1 个 CD 技能（顺位 4）
- 0~1 个能量技能（顺位 5）
- 0~1 个终结技能（顺位 6）

技能名称使用英文全拼，效果由 AI 自行设计，确保每个角色有独特的玩法。

### 12.4 领主技能模板
领主的技能作为参考模板（已在第十五行中给出示例）：
- 普通攻击：Horizontal Slash — 对面前横向 3 格敌人造成 1 倍伤害
- 移动：Infantry Move — 移动 4 格，无地形加成
- 技能点技能：Speed Boost — 增加自身速度条速度
- CD 技能：Inspire — 增加周围 9×9 范围内队友的速度（可插队）
- 能量技能：All-Out Charge — 增加全地图所有队友的速度（可插队）
- 终结技能：Double Time — 让自身速度翻倍（可插队，每场一次）

## 十三、分数系统

### 13.1 分数计算
- 每局游戏结束后，分数 = **本局造成的伤害总量**
- 包括所有战斗（子战斗）中造成的所有伤害之和
- 不包括过量伤害（伤害不超过目标剩余 HP）

### 13.2 经验值转换
- 分数按比例转换为经验值（XP）
- XP 用于局外养成系统

## 十四、战斗界面布局（圣杯式界面）

战斗中的界面采用**圣杯式（Grail Layout）** 布局，即中央大 + 四角信息面板。

```
┌──────────────────────────────────────────────────────────────┐
│  ┌──────────┐  ┌──────────────────────────────┐  ┌────────┐ │
│  │ SPEED    │  │                              │  │MINIMAP │ │
│  │  BAR     │  │        BATTLE MAP            │  │(全局)  │ │
│  │          │  │      (100×100 grid)          │  │        │ │
│  │  Unit A  │  │                              │  │        │ │
│  │  Unit B  │  │    (viewport showing         │  ├────────┤ │
│  │  Unit C  │  │     partial area)            │  │ SKILL  │ │
│  │  Unit D  │  │                              │  │ POINTS │ │
│  │  Unit E  │  │                              │  │ FOOD   │ │
│  │  Unit F  │  │                              │  │ POTION │ │
│  │  ...     │  │                              │  │        │ │
│  │          │  │                              │  ├────────┤ │
│  │          │  │                              │  │LEADER  │ │
│  │          │  │                              │  │ASCII   │ │
│  │          │  │                              │  │PORTRAIT│ │
│  └──────────┘  └──────────────────────────────┘  └────────┘ │
└──────────────────────────────────────────────────────────────┘
```

### 14.1 中央区域 — 战棋地图（Battle Map）

- 地图总大小为 **100 × 100 格**
- 每个格子显示为 **10 × 10 字符** 的正方形区域
- 由于地图太大，视口（viewport）只显示**当前可见部分**
- 地图本身是一个正方形
- 地图上显示：
  - 每个队伍的**位置**
  - 地形（障碍/水域/树林/平原等）
  - 每个队伍上方显示队伍名称缩写

### 14.2 左侧 — 速度条（Speed Bar）

- 显示所有单位的**速度条排序**
- 从上到下按速度从快到慢排列
- 当前轮到行动的单位**高亮显示**
- 显示每个单位的名称缩写和速度值

### 14.3 右上角 — 小地图（Minimap）

- 显示**全局地图**的缩略视图
- 标记所有队伍的位置
- 标记当前视口在全局中的位置（矩形框）

### 14.4 右下角 — 被选中队伍的队长 ASCII 头像

- 显示当前**被选中的队伍**的**队长的 ASCII 字符画头像**
- 注意：这是**被选中的那个队伍的队长**的头像画，不是固定的队长
- 使用 ASCII 艺术字符绘制

### 14.5 右上与右下之间 — 队伍资源面板

从上到下依次显示：

1. **技能点（Skill Points）**：全队共享的技能点数量
2. **食物栏（Food Bar）**：食物是可多次使用的消耗品，显示食物名称和剩余使用次数
3. **药水栏（Potion Bar）**：药水是单次消耗品，显示药水名称和数量

> **食物 vs 药水**：两者都是消耗品。食物可以**多次使用**（有使用次数），药水是**一次性**的。

### 14.6 队伍信息显示（按排位顺序）

当选中一个队伍时，在界面中显示该队伍的详细信息。**角色按排位顺序显示**（先 Front → 再 Mid → 再 Back，同一排位内按任意顺序）：

```
[Team: HEROES]
 Pos  | Name             | HP    | CD   | Energy | Ult  | Star
------+------------------+-------+------+--------+------+-------
 FRONT| Alexander        | 45/50 | 2/3  | 80/100 | Ready| ★★
 FRONT| Beatrice         | 30/40 | 0/2  | 50/100 | Not  | ★
 MID  | Christopher      | 60/60 | 1/4  | 100/100| Ready| ★★★
 MID  | Diana            | 20/35 | 3/3  | 10/100 | Not  | ★
 BACK | Edward           | 40/40 | 0/1  | 70/100 | Not  | ★★
 BACK | Fiona            | 50/50 | 2/5  | 90/100 | Ready| ★
```

每行显示：
- **排位（Position）**：Front / Mid / Back
- **角色全拼名字（Full Name）**：英文全拼
- **HP**：当前生命值 / 最大生命值
- **CD**：当前冷却 / 最大冷却（CD技能冷却状况）
- **Energy**：当前能量 / 100（能量技能充能状况）
- **Ult**：终结技是否可用（Ready / Not）
- **Star**：当前星级（★ 数量）

### 14.7 刷新机制

- **每次速度条轮到某个单位行动时，界面刷新一次**
- 其他时间界面**不刷新**
- 用户输入指令后，需要输入 **`continue`** 命令来推进到下一次刷新
- 操作流程：
  1. 界面显示当前状态
  2. 用户输入操作指令（移动/攻击/使用技能等）
  3. 用户输入 `continue`
  4. 界面刷新到下一个状态
  5. 重复
- 如果不进行操作，直接输入 `continue` 则跳过该单位行动
- 游戏是完全由用户控制的节奏型（turn-based），非实时

## 十五、战斗结算与抽卡系统

### 15.1 战斗结束后的奖励

每次战斗（包括小怪战斗）结束后，玩家获得以下奖励：

#### 抽卡（Gacha）
- 每次战斗结束后，玩家进行 **100 次抽卡**
- 抽卡获得**不同的角色**
- 角色**没有稀有度**之分
- 从 1000 个角色的池子中随机抽取

#### 装备掉落
- 每次战斗结束后掉落 **5 个装备**
- 装备**没有稀有度**，**没有星级**
- 装备可以重复获得

#### 消耗品掉落
- 战斗结束后可能掉落**药水**或**食物**

#### 遗物掉落
- 战斗结束后**可能**掉落遗物（稀有，不是每次都有）

### 15.2 角色升星系统

- 角色**没有稀有度**，所有角色都可以升星
- 升星方式：**2 合 1**
  - 2 个相同的 1 星角色 → 1 个 2 星角色
  - 2 个相同的 2 星角色 → 1 个 3 星角色
  - 以此类推
- **必须相同角色**才能合成
- 升星后的效果：
  - 所有技能得到**增幅**（伤害/治疗/效果等数值提升）
  - 在子战斗中，各项属性得到**全面增幅**
  - 星级在伤害公式中作为 `star_modifier` 参与计算

### 15.3 获取途径总览

| 物品类型 | 战斗掉落 | 商店 | 事件 |
|---------|---------|------|------|
| 角色 | ✅ 抽卡100次 | ✅ | ✅ |
| 装备 | ✅ 每次5个 | ✅ | ✅ |
| 药水 | ✅ 可能掉落 | ✅ | ✅ |
| 食物 | ✅ 可能掉落 | ✅ | ✅ |
| 遗物 | ✅ 可能掉落（稀有） | ✅ 可购买 | ✅ 可能获得 |

## 十七、开发约束

### 17.1 技术栈
- 语言：**Rust**
- 运行环境：任何支持 CLI 的系统（Windows CMD / PowerShell / Linux Terminal / macOS Terminal）
- 字符显示：等宽正方形字符（1/4 正常字符大小）

### 17.4 质量要求
- 代码完整可运行，无需后期手动调整
- 包含基础 AI（电脑对手的决策逻辑）
- 包含完整的局外养成系统
- 包含完整的存档/读档功能
- Rust 代码遵循标准风格（`cargo clippy` 无警告）
